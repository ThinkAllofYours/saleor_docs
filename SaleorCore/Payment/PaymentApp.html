결제 앱으로 결제 처리하기

  • 이 섹션에서는 결제 앱을 사용한 결제 처리에 대한 흐름을 설명합니다. 스토어프론트와 결제 앱 간의 통신은 Saleor를 통해 이루어집니다.
  • 결제 앱의 응답에 기반하여 Saleor는 적절한 상태로 트랜잭션을 생성하거나 업데이트합니다.

결제 앱을 초기화하려면 paymentGatewayInitialize 뮤테이션을 호출합니다. paymentGateways에 제공된 데이터는 결제 앱으로 전송됩니다. 그런 다음 Saleor는 각 앱에서 받은 데이터를 응답으로 반환합니다.

이 단계는 결제 앱에서 특정 체크아웃에 대해 사용 가능한 결제 수단을 가져오는 등의 데이터를 가져와야 할 때 유용합니다.

paymentGatewayInitialize 뮤테이션에는 다음 인수가 필요합니다:

  1. id - 체크아웃 또는 주문의 ID입니다.
  2. amount - 결제 게이트웨이를 초기화하기 위해 요청된 금액입니다. 제공되지 않은 경우, 체크아웃.total과 이미 처리된 트랜잭션 간의 차이가 앱으로 전송됩니다.
  3. paymentGateways - 초기화할 결제 게이트웨이 목록입니다. 제공되지 않은 경우, webhook PAYMENT_GATEWAY_INITIALIZE_SESSION에 가입한 모든 앱이 사용됩니다.
  4. paymentGateways.id - 초기화할 결제 게이트웨이 앱의 식별자입니다.
  5. paymentGateway.data - 결제 게이트웨이에 전달될 데이터입니다.

paymentGatewayInitialize 뮤테이션은 다음 응답을 반환합니다:

  1. gatewayConfigs - 초기화된 결제 앱 목록입니다.
  2. gatewayConfigs.id - 앱 식별자입니다.
  3. gatewayConfigs.data - 앱에서 반환한 결제 게이트웨이를 초기화하는 데 필요한 데이터입니다.
  4. gatewayConfigs.errors - 특정 결제 게이트웨이 앱과 관련된 오류 목록입니다.
  5. errors - 뮤테이션 호출과 관련된 오류입니다.
mutation {
  paymentGatewayInitialize(
    id: "Q2hlY2tvdXQ6ZmIxMzljMjgtYWY4OS00Mzk2LWEyMjgtZmM2ZDg0NGFhOWY3"
    amount: 100
    paymentGateways: [
      { id: "payment.app", data: { details: { passed: "to-app" } } }
    ]
  ) {
    gatewayConfigs {
      id
      data
      errors {
        field
        message
        code
      }
    }
    errors {
      field
      message
      code
    }
  }
}
  • response 는 아래와 같습니다.
{
  "data": {
    "paymentGatewayInitialize": {
      "gatewayConfigs": [
        {
          "id": "payment.app",
          "data": {
            "json": "data-returned-by-app"
          },
          "errors": []
        }
      ],
      "errors": []
    }
  }
}

뮤테이션은 PAYMENT_GATEWAY_INITIALIZE_SESSION 웹훅을 트리거합니다.

거래 초기화

결제 처리를 시작하려면 transactionInitialize 뮤테이션을 호출합니다. paymentGateways 필드에서 제공하는 데이터가 결제 앱으로 전송되며, Saleor는 앱에서 받은 데이터를 응답으로 반환합니다. 결제 앱의 응답에 따라, Saleor는 적절한 상태로 거래를 생성하거나 업데이트합니다.

transactionInitialize 뮤테이션에는 다음 인수가 필요합니다
  • id - 체크아웃 또는 주문 ID.
  • paymentGateway - 결제 처리를 시작하는 데 사용되는 결제 게이트웨이.
  • paymentGateway.id - 결제 게이트웨이 앱의 식별자.
  • paymentGateway.data - 결제 게이트웨이에 전달할 데이터.
  • action - 거래에 대해 호출되는 예상 액션. 기본적으로 channel.defaultTransactionFlowStrategy가 사용됩니다. 이 필드는 HANDLE_PAYMENTS 권한이 있는 앱에서만 사용할 수 있습니다.
  • amount - 결제에 요청된 금액. 제공되지 않은 경우, checkout.total과 이미 처리된 거래 간의 차이가 전송됩니다.
transactionInitialize 뮤테이션은 다음 응답을 반환합니다
  • transaction - 초기화된 거래.
  • transactionEvent - 결제 앱에서 받은 응답을 기반으로 생성된 이벤트.
  • data - 앱에서 반환된 데이터.
  • errors - 뮤테이션 호출과 관련된 오류.

뮤테이션은 TRANSACTION_INITIALIZE_SESSION 웹훅을 트리거합니다.

다음 예제는 transactionInitialize 뮤테이션을 사용하여 결제 처리를 시작하는 방법을 보여줍니다:

mutation TransactionInitialize {
  transactionInitialize(
    amount: 100
    id: "Q2hlY2tvdXQ6MzE3NDk5MjgtZDFkMC00NjdjLTgxNjktMGRiZGM4ZGJhMGZh"
    paymentGateway: { id: "payment.app", data: { details: "passed-to-app" } }
  ) {
    transaction {
      id
    }
    transactionEvent {
      type
      pspReference
    }
    errors {
      field
      message
      code
    }
  }
}

The response would look like this:

{
  "data": {
    "transactionInitialize": {
      "data": {
        "some-json": "data"
      },
      "transaction": {
        "id": "VHJhbnNhY3Rpb25JdGVtOjhhMzE1ODVlLWE1MTktNDVkMS1hNTFmLTYwZGZlNGRkM2FkNg"
      },
      "transactionEvent": {
        "type": "CHARGE_SUCCESS",
        "pspReference": "ppp-123"
      },
      "errors": []
    }
  }
}

거래 처리

transactionInitialize 또는 transactionProcess가 transactionEvent.type을 AUTHORIZATION_ACTION_REQUIRED 또는 CHARGE_ACTION_REQUIRED와 동일하게 반환하면 고객의 추가 조치가 필요합니다. 추가 조치의 결과를 앱에 제공하려면 transactionProcess 뮤테이션을 호출합니다.

transactionProcess 뮤테이션에는 다음 인수가 필요합니다:
  • id - transactionInitialize에서 반환된 거래 ID.
  • data - 결제 게이트웨이에 전달할 데이터.
transactionProcess 뮤테이션은 다음을 반환합니다:
  • transaction - 처리된 거래.
  • transactionEvent - 결제 앱에서 받은 응답을 기반으로 생성된 이벤트.
  • data - 앱에서 반환된 데이터.
  • errors - 뮤테이션 호출과 관련된 오류.

뮤테이션은 TRANSACTION_PROCESS_SESSION 웹훅을 트리거합니다.

다음 예제는 transactionProcess 뮤테이션을 사용하여 결제를 처리하는 방법을 보여줍니다:

mutation TransactionInitialize {
  transactionProcess(
    id: "VHJhbnNhY3Rpb25JdGVtOjhhMzE1ODVlLWE1MTktNDVkMS1hNTFmLTYwZGZlNGRkM2FkNg"
    data: { additional: { actions: "details" } }
  ) {
    transaction {
      id
    }
    transactionEvent {
      type
      pspReference
    }
    errors {
      field
      message
      code
    }
  }
}

The response would look like this

{
  "data": {
    "transactionProcess": {
      "data": {
        "some-json": "data"
      },
      "transaction": {
        "id": "VHJhbnNhY3Rpb25JdGVtOjhhMzE1ODVlLWE1MTktNDVkMS1hNTFmLTYwZGZlNGRkM2FkNg"
      },
      "transactionEvent": {
        "type": "CHARGE_SUCCESS",
        "pspReference": "ppp-123"
      },
      "errors": []
    }
  }
}
결제의 현재 상태는 transactionEvent.type의 값에 따라 결정할 수 있습니다:
  • AUTHORIZATION_ACTION_REQUIRED 또는 CHARGE_ACTION_REQUIRED는 고객이 결제를 완료하기 위해 추가 조치를 요구한다는 것을 나타냅니다. action의 결과를 앱에 전달하려면 transactionProcess를 호출해야 합니다.
  • AUTHORIZATION_SUCCESS는 자금이 승인되었음을 나타냅니다.
  • AUTHORIZATION_FAILURE는 인증이 실패했음을 나타냅니다.
  • AUTHORIZATION_REQUEST는 인증이 보류 중인 상태임을 나타냅니다.
  • CHARGE_SUCCESS는 자금이 청구되었음을 나타냅니다.
  • CHARGE_FAILURE는 청구가 실패했음을 나타냅니다.
  • CHARGE_REQUEST는 청구가 보류 중인 상태임을 나타냅니다.
