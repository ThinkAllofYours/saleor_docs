checkout 앱을 사용한 결제 처리

이 섹션에서는 Checkout과 같은 외부 Saleor 앱에서 결제 프로세스가 처리되는 흐름에 대해 설명합니다. 이 경우, Saleor는 Saleor 앱에서 처리된 결제 트랜잭션의 결과를 받기를 기대합니다.

트랜잭션은 주문이나 체크아웃에 연결된 결제 트랜잭션의 세부 정보를 저장합니다.

transactionCreate 뮤테이션은 다음 인수를 사용합니다:
  • id: 체크아웃이나 주문의 ID입니다.
  • transaction: 새로운 트랜잭션 객체를 생성하는 데 필요한 입력 데이터입니다.
  • transactionEvent: 트랜잭션 이벤트를 정의하는 데이터입니다. 트랜잭션의 현재 상태에 대한 맥락을 제공하는 데 사용할 수 있습니다.

다음 예시는 transactionCreate 뮤테이션을 사용하여 새로운 트랜잭션을 생성하는 방법을 보여줍니다.

트랜잭션이 승인되었고, 신용카드로 결제가 이루어졌습니다. Saleor에서 호출할 수 있는 액션은 CANCEL과 CHARGE입니다. 승인된 금액은 $99입니다.

mutation {
  transactionCreate(
    id: "Q2hlY2tvdXQ6MWQzNmU5YzctYWEwYS00NzM5LTk0MGQtNzdjNmU4Mjc5YmQ0"
    transaction: {
      status: "Authorized"
      type: "Credit card"
      reference: "PSP-ref123"
      availableActions: [CANCEL, CHARGE]
      amountAuthorized: { currency: "USD", amount: 99 }
    }
    transactionEvent: {
      status: SUCCESS
      name: "Authorized credit card"
      reference: "PSP-ref123"
    }
  ) {
    transaction {
      id
    }
  }
}

The response would look like this:

{
  "data": {
    "transactionCreate": {
      "transaction": {
        "id": "VHJhbnNhY3Rpb25JdGVtOjE="
      }
    }
  },
  "extensions": {
    "cost": {
      "requestedQueryCost": 0,
      "maximumAvailable": 50000
    }
  }
}

트랜잭션 업데이트

transactionUpdate 뮤테이션을 사용하면 트랜잭션 세부 정보를 업데이트할 수 있습니다. 다음 인수를 사용합니다:

  • id: 트랜잭션의 ID입니다.
  • transaction: 트랜잭션 객체를 업데이트하는 데 사용되는 입력 데이터입니다.
  • transactionEvent: 트랜잭션 이벤트를 정의하는 데이터입니다. 트랜잭션의 현재 상태에 대한 맥락을 제공하는 데 사용할 수 있습니다.

transactionUpdate는 트랜잭션 소유자만 호출할 수 있습니다. 소유자는 해당 트랜잭션을 생성한 사용자 또는 앱입니다.

다음 예시는 트랜잭션을 업데이트하기 위해 transactionUpdate 뮤테이션을 사용하는 방법을 보여줍니다.

사용 가능한 액션은 REFUND입니다. 인증된 자금이 청구되었으므로 amountAuthorized는 $0입니다. amountCharged는 $99와 같습니다.

mutation {
  transactionUpdate(
    id: "VHJhbnNhY3Rpb25JdGVtOjE="
    transaction: {
      name: "Credit card"
      message: "Authorized"
      pspReference: "PSP-ref123"
      availableActions: [REFUND]
      amountAuthorized: { currency: "USD", amount: 0 }
      amountCharged: { currency: "USD", amount: 99 }
    }
    transactionEvent: {
      message: "Payment charged"
      pspReference: "PSP-ref123.charge"
    }
  ) {
    transaction {
      id
    }
  }
}

The response would look like this:

{
  "data": {
    "transactionUpdate": {
      "transaction": {
        "id": "VHJhbnNhY3Rpb25JdGVtOjE="
      }
    }
  },
  "extensions": {
    "cost": {
      "requestedQueryCost": 0,
      "maximumAvailable": 50000
    }
  }
}

트랜잭션 업데이트 중에는 새 상태로 이동하는 모든 자금이 이전 상태에서 차감되어야 합니다. authorizedAmount가 100 USD인 트랜잭션을 가정해 봅시다. authorizedAmount를 chargedAmount로 이동하려면 authorizedAmount를 0으로 설정해야 합니다.

mutation {
  transactionUpdate(
    id: "VHJhbnNhY3Rpb25JdGVtOjE="
    transaction: {
      status: "Charged"
      availableActions: [REFUND]
      amountAuthorized: { currency: "USD", amount: 0 }
      amountCharged: { currency: "USD", amount: 100 }
    }
    transactionEvent: {
      status: SUCCESS
      name: "Charged credit card"
      reference: "PSP-ref123.charge"
    }
  ) {
    transaction {
      id
    }
  }
}
