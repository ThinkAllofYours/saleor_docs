Configuration

Environment variables

  • SRS는 Next.js 프레임워크를 사용하여 구축됩니다. 환경 변수를 사용한 작업에 대한 완전한 워크플로가 포함되어 있습니다. 이에 대해 문서에서 자세히 알아볼 수 있습니다.

  • 환경 변수를 통해 변경할 수 있는 모든 옵션은 .env 파일에서 찾을 수 있습니다.

Example - Changing the API URI

  • 로컬 개발의 경우, .env.local 파일을 통한 구성을 추천합니다:

  • 프로젝트의 루트에 .env.local를 생성합니다.

  • NEXT_PUBLIC_API_URI=http://localhost:8000/graphql/ 항목을 추가합니다.

  • 개발 서버가 실행 중이었다면, 서버 재시작 후 새 값이 로드됩니다.

  • Git은 .env.local 파일을 무시하여 로컬 구성이 노출되지 않도록 합니다.

  • 애플리케이션 배포의 경우, 파일 대신 환경 변수를 설정하는 것이 좋습니다. 설정 방법은 호스팅 공급자에 따라 다릅니다. Vercel을 사용하는 경우, 관련 문서는 여기에서 찾을 수 있습니다.

Changing available channels

  • 사용 가능한 채널 목록은 lib/regions.ts 파일에 위치한 CHANNELS 상수에 새 항목을 추가함으로써 변경할 수 있습니다.

Languages

  • 사용 가능한 언어를 구성하려면 이 문서를 참고하세요 read translation docs.

Homepage

  • 홈페이지는 기본적으로 API에서 가져온 Menu 객체에 기반한 콘텐츠 블록을 표시합니다. 객체의 슬러그는 NEXT_PUBLIC_HOMEPAGE_MENU 변수를 사용하여 구성할 수 있습니다.
