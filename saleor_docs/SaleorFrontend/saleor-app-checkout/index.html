Setup

Prerequisites

monorepo의 루트에 있는 SALEOR_API_URLSALEOR_API_URL 이 Saleor 인스턴스로 설정되어 있는지 확인하세요. monorepo 루트의 .env서 정의된 Saleor 인스턴스 외에도 다른 Saleor 인스턴스를 사용할 수 있습니다.

Development

Run the development server:

pnpm dev

  • saleor-app-checkout 앱을 로컬에서 개발하려면 터널을 만들어야 합니다. 터널은 Saleor 대시보드 내에서 표시할 수 있게 해줍니다. Saleor 앱 아키텍처에 대해 자세히 알고 싶다면 문서를 참조하세요.

    the documentation.

  • To create a tunnel, run:

    npx saleor app tunnel 3001

  • local에서 개발할 때는 ngrok 를 이용해서 saleor-app-checkout 이 실행되고 있는 3001번 포트에 ngrok를 연결하고 http://localhost:9000/dashboard/apps/에서 install external app을 통해 내 로컬 localhost:3001을 ngrok에 연결함

ngrok http 3001
  • 등록주소 예시

    https://8fd3-58-124-34-94.jp.ngrok.io/api/manifest/

  • 여기서 3001은 saleor-app-checkout이 기본적으로 실행되는 포트입니다. 터널을 만드는 대체 방법은 아래에 설명되어 있습니다.

Note
--force-install인수는 처음 명령을 실행할 때만 필요합니다. 이를 통해 Saleor 대시보드 내에 애플리케이션을 설치하고 토큰을 저장합니다.

saleor app tunnel 에서 받은 터널 URL(예: https://saleor-app-checkout-xyz-my-org.saleor.live)을 사용하여 브라우저에서 결과를 확인합니다.

Warning
터널은 백그라운드에서 실행되어야 합니다. 개발 중에는 종료하지 마세요.

Note
인터넷 연결이 변경되면 터널이 끊어질 수 있습니다. 터널에서 앱에 액세스할 수 없으면 다시 시작하십시오.

Alternative way to obtain the token

토큰은 다음 CLI 명령을 사용하여 생성할 수도 있습니다:

npx saleor app token

이제 .saleor-app-auth.json이라는 이름의 빈 JSON 파일을 만들고 토큰을 붙여넣습니다. 파일은 다음과 같아야 합니다:

{
  "token": "(your application's auth token)",
  "domain": "(your Saleor GraphQL API URL)"
}

Production

프로덕션용으로 빌드하려면 다음 명령을 실행합니다:

cd ../.. && pnpm run build:saleor-app-checkout

Note
명령은 monorepo의 루트에서 실행되어야 합니다.

배포된 앱은 환경 변수를 사용하여 수동으로 인증 토큰을 설정해야 합니다. Saleor에 앱을 설치하고 Saleor CLI를 사용하여 토큰을 생성합니다:

saleor app token
  • 받은 토큰 값을 SALEOR_APP_TOKEN 환경 변수 값으로 설정하고 앱을 다시 배포합니다.

Other

saleor app tunnel 대체 방법

  • You can also use ngrok,

  • 하지만 매번 터널을 열 때마다(무료 플랜에서) ngrok이 할당한 새 도메인으로 환경 변수를 업데이트해야 합니다.

  • 그러한 작업 후에는 saleor app install을 사용하여 Saleor 인스턴스에 앱을 설치할 수 있습니다.

  • 터널은 유효한 SSL 인증서가 있는 https 프로토콜을 사용해야 하며, 그렇지 않으면 웹훅 호출을 받지 못합니다.

Env Variables

환경 변수에는 스토어 데이터를 위험에 빠뜨릴 수 있는 비밀이 포함되어 있습니다

When deploying to Vercel you can set them on the configuration page

When running app locally you can use the .env.local file. It should not be included in your git repository.

Backend variables

  • SALEOR_APP_TOKEN — 앱이 설치된 후 Saleor에서 생성된 앱 토큰
  • SETTINGS_ENCRYPTION_SECRET — 앱 구성을 암호화하는 데 사용되는 무작위 문자열

무작위 비밀을 생성하려면 OpenSSL을 사용할 수 있습니다:

openssl rand -hex 256

Learn more 세일러 문서에서 제3자 앱 설치에 대해 알아보십시오 Saleor docs

  • DEBUG_APP_URL - 배포된 체크아웃 앱의 URL입니다. 로컬에서 앱을 실행하고 saleor tunnel을 사용하여 요청을 터널링할 때 사용됩니다.

Warning
이 변수는 로컬 개발에만 사용되어야 합니다! 프로덕션 배포에서는 무시됩니다.

프로덕션에서는 각 요청의 Host 헤더에서 이 URL이 가져옵니다.

Frontend variables

NEXT_PUBLIC 으로 시작하는 각 변수는 프론트엔드에 노출됩니다.

Testing

React 컴포넌트를 테스트하려면 jsdom 런타임을 사용해야 합니다(기본값은 node). 테스트 파일의 맨 위에 이 주석을 추가하십시오:

/** @jest-environment setup-polly-jest/jest-environment-node */

Mocking requests

Mocking configuration는 Node 환경에서 모두 작동합니다.

Recording

  • 테스트에서 이루어진 요청을 모의하기 위해 Polly.js.를 사용합니다. 기본적으로 테스트에서 요청이 이루어지고 모의되지 않으면 테스트가 실패합니다.

요청을 녹음하려면 test:record 명령을 사용하여 테스트를 실행하십시오

pnpm run test:record

이 명령은 POLLY_MODE 환경 변수를 record로 설정합니다.

녹음된 내용은 recordings 폴더에 저장됩니다.

Manual mocks

수동 모의를 작성하려면 Mock Service Worker라이브러리를 사용합니다. msw를 사용하여 모의된 REST API 라우트 또는 GraphQL 작업은 Polly.js에 의해 녹음되지 않습니다. msw용 모의는 mocks/handlers에 위치해 있습니다.

Checkout Storefront

체크아웃 스토어프론트는 []/checkout-spa/?saleorApiUrl=] ../saleor-app-checkout/pages/checkout-spa.tsx에서 이용할 수 있습니다.

체크아웃을 사용하려면 토큰이 필요합니다. 새로운 체크아웃 세션은 스토어프론트 또는 GraphQL Playground에서 생성할 수 있습니다. 기존 체크아웃을 사용할 수도 있습니다.

Warning
주어진 체크아웃에 이미 고객이 연결되어 있다면, 해당 체크아웃은 비공개가 되고, 현재 브라우저에서 동일한 고객이 로그인되어 있지 않으면 API에서 데이터를 가져올 수 없게 됩니다. 체크아웃은 Saleor SDK를 인증에 사용합니다. GraphQL API에서 체크아웃을 생성하고 id를 검색하려면 다음을 실행합니다:

mutation {
  checkoutCreate(
    input: {
      channel: "default-channel"
      lines: [{ variantId: "UHJvZHVjdFZhcmlhbnQ6MjE0", quantity: 1 }]
    }
  ) {
    checkout {
      id
    }
  }
}

체크아웃 세션 생성에 대한 자세한 내용은 Saleor 문서에서 확인하십시오.여기 Saleor docs http://localhost:3001/checkout-spa/?saleorApiUrl=<YOUR_SALEOR_API_URL>&checkout=<TOKEN>을 복사하고, YOUR_SALEOR_API_URL을 Saleor GraphQL 엔드포인트의 URL로 대체하고, TOKEN을 체크아웃 세션의 토큰으로 대체한 다음 브라우저에서 엽니다.
