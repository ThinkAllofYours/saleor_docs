Translations

두 가지 종류의 번역이 있습니다:

  • 대시보드를 통해 관리되고 API에서 가져올 수 있는 번역입니다. 예를 들어, 특정 상품의 이름과 같은 것입니다.
  • 인터페이스 번역입니다. 예를 들어, "장바구니에 추가" 버튼의 라벨입니다

API translations

  • API에 대한 문서는 여기에서 찾을 수 있습니다. here.

  • 대시보드에 대한 가이드는 여기에서 찾을 수 있습니다. here.

  • React Storefront에는 lib/translations.ts 파일에 위치한 translate라는 이름의 번역된 필드를 표시하는 도우미가 있습니다. 이 함수는 API에 번역이 없는 경우 기본 언어로 돌아갑니다.

Interface translations

  • SRS는 인터페이스 번역을 위한 라이브러리로 FormatJS를 사용합니다. 이전에 사용해보지 않았다면 문서를 읽어보시는 것이 좋습니다.documentation

  • 번역 소스는 locale 디렉터리에 위치해 있습니다. SRS는 언어 방언을 구별하기 위해 전체 언어 코드 이름을 사용합니다(예: en-US 및 en-UK).

Adding new languages

  1. locale 디렉터리에 새로운 json 파일을 생성합니다.
  2. lib/regions.ts 파일의 LOCALES 상수에 언어를 추가합니다.
  3. 선택한 로케일에 기반하여 적절한 json 파일을 가져오도록 importMessages 함수를 확장합니다.

Translation management

파일을 수동으로 편집하는 대신 번역 플랫폼을 사용하는 것이 좋습니다:

  • Transifex
  • POEditor

다른 여러 온라인 플랫폼과 로컬로 설치된 소프트웨어 중에서 선택할 수 있습니다. 유일한 요구 사항은 JSON 파일을 동기화하는 것입니다.
