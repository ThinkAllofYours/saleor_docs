Saleor 팀에서 제공하는 오픈소스 프론트엔드 전자상거래 플랫폼.

Development

GraphQL queries

  • GraphQL 쿼리는 ./graphql 폴더에 위치하고 있습니다. fragments 사용을 강력히 권장하며, 이를 통해 코드 중복을 최소화하고 TypeScript로 들어오는 데이터 변환과 잘 호환됩니다.

  • 우리의 선택된 클라이언트는 Apollo로, 기본 제공되는 훌륭한 캐시 및 기능을 제공합니다. 완전하게 타입화된 요청과 응답을 얻기 위해, GraphQL Code Generator가 모든 .graphql 파일을 사용할 준비가 된 훅으로 변환합니다. 생성된 코드는 ./saleor/api.tsx 파일에 위치하고 있습니다.

  • API 엔드포인트는 문서에서 설명한 대로 .env 파일을 통해 설정할 수 있습니다.

Workflow

  1. GraphQL 파일 수정 또는 생성방법
    • 예를 들어 ./graphql/queries/FeaturedProducts.graphql에 새 쿼리 추가
  2. pnpm generate 명령 실행
  3. 새 쿼리가 ./saleor/api.tsx 파일에 추가됩니다.
  4. 컴포넌트 코드에서 생성된 훅 임포트 import { useFeaturedProductsQuery } from "@/saleor/api";

React and Next.js code structure

  • 컴포넌트를 새롭게 생성할때는 이 링크에 적혀 있는 지침을 따라생성 하시면 됩니다. React TypeScript Cheatsheet.
  • 결제 게이트웨이 코드는 ./components/checkout/payments에서 찾을 수 있습니다. 현재 Saleor 테스트 게이트웨이와 Stripe의 기본 흐름을 지원합니다.

Routing and urls

  • 프로젝트는 파일 기반 라우팅을 사용합니다. 사용 가능한 라우트는 ./pages에서 찾을 수 있습니다.

    동적 라우트 (예: ./pages/product/[slug].tsx)는 getStaticPaths를 기반으로 빌드 시간에 생성됩니다.

  • Link 컴포넌트가 필요한 인수와 함께 기존 URL만 사용하도록 보장하기 위해 pathpida를 사용합니다. 이를 사용하여 ./lib/$path.ts 파일에 모든 사용 가능한 라우트가 자동으로 생성됩니다. 파일을 수동으로 업데이트하지 않고 대신 실행하세요:

pnpm paths
  • 라우트가 현재 로케일 및 채널과 함께 추가 인수를 필요로 하므로, 자동으로 추가하는 usePaths 훅을 사용해야 합니다. 제품 페이지로의 링크가 있는 예제 컴포넌트를 만들어 보겠습니다
import Link from "next/link";
import { usePaths } from "@/lib/paths";

export const ProductLinkComponent = () => {
  const paths = usePaths();
  return (
    <Link href={paths.products._slug(line?.variant?.product?.slug).$url()}>
      <a>Product link</a>
    </Link>
  );
};

Bundle metrics

or, configure it directly in the respective cloud provider

Heroku

  • Heroku의 경우, package.json에 명시적으로 LTS 버전의 Node.js를 지정해야 합니다. package.json에 다음 스니펫을 추가하세요:
"engines": {
  "node": ">=14 <17",
  "npm": ">=6.11.0 <8"
}

이 코드베이스에서 최신 Node.js 버전을 대상으로 하기 때문에 이것을 추가하지 않습니다.
